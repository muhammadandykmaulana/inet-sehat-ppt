INTERNET SEHAT

DNS Content Blocker & Crowdsourced Reporting

"Menciptakan Koneksi Internet yang Lebih Aman dan Sehat di Level Jaringan Tanpa Instalasi Aplikasi Tambahan di Perangkat."

Disusun Oleh: Muhammad Andyk Maulana

Latar Belakang & Solusi

Ancaman Konten Ilegal

Akses ke situs iklan intrusif, pornografi, dan perjudian online di jaringan institusi/rumah semakin tak terkendali.

Kelemahan Cara Lama

Pemblokiran via ekstensi browser mudah dimatikan oleh user dan tidak praktis karena harus diinstal per-perangkat.

Solusi: DNS Filtering

Memblokir langsung dari hulu (Router/DNS). Satu konfigurasi, seluruh perangkat dalam jaringan otomatis terlindungi.

Pilar Teknologi & API

Arsitektur mandiri yang dibangun tanpa bergantung pada heavy-framework berlebih.

Backend

Python 3 & Flask. Menangani Routing, API Internal, Blueprint, dan Templating Jinja2.

Network

Custom DNS Server via Python Socket. Gateway HTTP internal untuk merender halaman blokir.

Database

SQLite3. File-based, ringan, cepat, tanpa instalasi server database eksternal.

API & Blocklist

Integrasi Whoisjson API untuk skoring risiko (usia domain). Auto-update list dari AdGuard & EasyList.

Cara Setup & Deployment Mudah

Proyek ini dilengkapi dengan bash script (setup.sh) untuk deployment 1-klik di server Linux.

# 1. Clone repository ke server VPS Anda
git clone https://github.com/muhammadandykmaulana/koneksaun-saudavel.git
cd koneksaun-saudavel

# 2. Jalankan script setup otomatis
# Script akan otomatis membuat virtual environment, install PIP,
# Inisialisasi Database SQLite, dan setup systemd daemon.
sudo bash deploy/setup.sh

# 3. Verifikasi service (DNS & Web Flask) berjalan normal
sudo systemctl status ks-dns ks-web

Keamanan & Otorisasi

Autentikasi (Session)

Password di-hash menggunakan SHA-256 dengan kombinasi Salt Acak (16 byte) dan Pepper Statis. Mencegah serangan Rainbow Table.

Otorisasi (RBAC)

Menggunakan Custom Middleware Decorator @login_required dan @admin_required untuk membatasi aksi API dan View.

Error Handling

Respon JSON API terstandarisasi (400, 401, 403, 404, 500) dan pola Try/Except di layer Database untuk menjaga stabilitas *uptime*.
